The way was long and winding, but I got there.

  • 16 September

    I come up with my initial idea: to build a Large Language Model (LLM) similar to ChatGPT, capable of writing essays from given prompts. It's a very ambitious idea and I eagerly await to work on it.

  • 17 September

    After some more thinking, I find that this idea is very unrealistic. Learning about natural language processing, which takes a lot of time to implement correctly, AND learning about generation which is a whole other field of machine learning, AND finding the required datasets, AND doing it all well while documenting the entire process and coping with the high workload I have from other courses is definitely unachievable. So I switch my idea to image classification, which is much more feasible. It's also a rather standard machine learning problem, but I add a twist, whereby I predict the type of object instead of the precise object.

  • 25 September

    I find a dataset called "Caltech 256" that consists of 30000 images subdivided into 256 categories, which I consider good for my project. By now I have also started researching about the best models for image classification purposes. I quickly find about the convolutional neural network (CNN), a type of artificial neural network that incorporates convolutions before the fully connected part as well. More on that later.

  • 6 October

    Work has begun! I set up my computer with all the necessary libraries to create my product. Specifically, Jupyter Notebook - a tool that allows one to combine code snippets with written text, and libraries such as Numpy and TensorFlow for data analysis and creation of neural networks.

  • 14 October

    I scrap the Caltech 256 dataset in favor of a much more uniform and standardized dataset called CIFAR-100, which consists of 60000 images subdivided into 100 categories, which are grouped into 20 higher-level categories. Those categories will form the basis of determining the "type of object" present in an image, e.g. vehicles, land animals, flowers, etc.

  • 27 October

    Success criteria have now been formulated. It is set in stone in my process journal what exactly this project aims to achieve — a model that is accurate but also generalizable. By now research has reached a level where I feel confident in my understanding of machine learning as well as deep learning, at least up to a basic level.

  • 19 November

    The model hits 35% accuracy on the validation set, which is higher than it had ever been before.

  • 23 November

    My supervisor, Mrs. Rai, kindly arranged a meeting with an expert in machine learning. The meeting lasted roughly half an hour and it helped me identify some things wrong with my method, which helped in the short term. In the meeting, themes such as a model's overfitting and exploratory data analysis were discussed.

  • 5 December

    I had a second primary source meeting because of more technical issues related to my model's overfitting. A decision was made to increase the size of the dataset to 19500 images. I was also introduced to Google Colab, a platform which allows one to run code virtually on a GPU. It greatly increased my productivity in the final parts of my project.

  • 6 December

    And now begins a new era — one where I will be able to push forward with much more effective models than before. I discover the technique of batch normalization, which standardizes data to reduce the influence of outliers, and things get better — like, much better. Now my model can predict 55% of validation samples correctly. Absolutely brilliant.

  • 11 December

    The deadline inches ever closer... I try many different architectures and stumble upon one that works very well, and can generalize to unseen data. It's the one being used on this website. This day concludes the making process, although much work remains to be done.
